Controlled Structural Edits

Stage 3
User guide

Using Controlled Edits

Controlled Edits is a guarded way to change model structure. Edits are collected into a proposed transaction, checked on a private copy of the Canvas graph, and applied to the shared model only after the complete transaction is accepted and you choose Apply to Canvas.

1. Choose an edit

Add or remove a node or edge. Complete the displayed fields, then select Queue edit. Nothing on Canvas changes while an edit is only queued.

2. Build one transaction

Queue related edits together, such as adding a node and its connecting edges. Later queued fields can reference nodes and edges proposed earlier in the same transaction.

3. Set preservation rules

Protect important nodes or edges and add causal paths that must remain reachable. These rules supplement the always-active anti-collapse constraints.

4. Check, review, apply

Check transaction validates the entire proposal. Review its reason, reward, graph-size summary, and preservation score. An accepted check exposes Apply to Canvas.

Edit reference

Add nodeSets name, starting value, retention, optional bounds, and optional value/source/sink formulas. A formula converter must use retention 0.
Remove nodeRemoves the node and its incident edges. The full transaction must still satisfy every safety constraint.
Add edgeSets source, target, correlation, decay, confidence, delay, and functional form. New self-loops and duplicate edges are rejected.
Remove edgeRemoves the selected directed connection, provided protected elements and required causal paths remain intact.

Always-active safeguards

  • Retain at least 50% of the original nodes.
  • Retain at least 25% of the original edges.
  • Do not increase graph fragmentation or create isolated nodes.
  • Do not introduce new self-loops.

Important behavior

A rejected transaction does not change Canvas. If Canvas changes after a successful check, the preview becomes stale and must be checked again. Applying is rollback-safe: if replay fails, the original model is restored.

Recommended workflow: protect critical nodes and paths first, queue the smallest coherent set of edits, check it, read any rejection reason, and apply only when the preview matches your intent.

Team Workspace

Stage 5
User guide

Using Team Workspace

Team Workspace lets role-based teams test changes against the model currently open on Canvas. It uses the same graph data and simulation logic. A normal session remains a preview until you select Apply final to Canvas or keep the final replay frame.

1. Prepare the model

Build or load a connected model on Canvas first. Clear node names and meaningful starting values make objectives and results easier to interpret.

2. Define each team

Start from a role preset or use Custom. Choose what node the team owns and which node defines its objective. Behavior objectives target a dynamic class; trajectory objectives target a sequence of values. An optional maximum spectral radius adds a whole-model linear-stability goal.

3. Limit its authority

Permissions determine which parameters or structural edits a team may use. Budgets and move costs discourage excessive changes. Objective scope restricts actions to the objective node and adjacent edges.

4. Choose an approach

Greedy baseline picks the best immediate move. Learned actor-critic practices across repeated rounds and estimates longer-term value before the final session.

Color roles: Green and Blue are cooperative stabilizers. Red and Black are bounded adversaries. A disruptor still targets a healthy state in the form because the reward engine reverses that score. Presets fill editable settings; they do not bypass permissions, costs, budgets, or anti-collapse checks. Autonomous turns search bounded parameter changes and permitted structural motifs, including mediated paths, balancing links, and safe edge removal. Controlled Edits remains available when you want to queue an exact transaction yourself.

Learned-agent settings

Practice roundsHow many training episodes the selected learner receives. Start with 10–30.
Moves per roundMaximum learner decisions in each practice episode. Start with 5–8.
Learning windowHow many future steps contribute to each learning update. Five is a practical default.
Frozen opponentsHow non-learning teams behave during practice: hold, act randomly, or use a fixed greedy policy.
Planning depthHow far the learned agent looks ahead during the final session. Use 1 first; 2–3 take progressively longer.
Goal safeguardFinal learned moves are checked with the canonical simulator. If a stabilizer stalls before an explicit spectral target, the bounded planner supplies the next permitted retention or decay step; it cannot bypass team permissions.
Comparison seedsRuns random, greedy, and learned policies on matching scenarios. Three seeds is useful for a first comparison.
Maximum spectral radiusOptional whole-model limit from the notebook transition matrix. Values below 1 are linearly stable; the reward still keeps activity, transmission, and move-cost safeguards.
Structural budgetCounts individual node and edge edits. Inserting a node between two existing nodes uses four edits: add node, remove old edge, and add two replacement edges.
Balance current modelBuilds a bounded retention/decay plan for a connected model. Two-node balancing applies directly; larger-model plans stay in preview until you review the change table and select Apply final to Canvas.

Reading the results

  • Return: the accumulated objective score; higher is better for the defined team objective.
  • Learning curve: return by practice round. Some noise is normal.
  • Comparison: seed-matched averages for random, greedy, and learned decisions.
  • Exploitability proxy: how much a team could gain by switching policy. Lower is better, but it is not proof of equilibrium.

Replay and keep changes

Use Apply final to Canvas to commit the result directly. Use Replay on Canvas to inspect each move first. Exit restores the graph that was open before replay, while Keep final commits the last frame.

Recommended first run: 10 practice rounds, 5 moves, depth 1, no comparison seeds. Once that succeeds, use 30 rounds and 3 comparison seeds.

Balance the current model

Choose a whole-model spectral limit and which bounded parameters may change. The planner prioritizes influential nodes and edges, preserves structure and polarity, and checks the result with the full notebook simulation.
